# 施策蓄積サイト（テンプレート）

CRO施策の勝ち/引き分け/負けと「なぜそうなったか」を蓄積し、**次のデザイン提案の根拠**にするための社内向け静的サイトです。**事業部ごとに独立したリポジトリで運用**します。

## これは何のテンプレートか

- 施策をカードUIで一覧（勝ち/引き分け/負け）
- 横断インサイト（勝率・パターン・外れた仮説）
- 検証用シートPDFをドロップ → カードの下書きを自動生成
- 事実 と AI推論 を明示的に区別、デザイン観点（✅/⚠️/❌）を蓄積

データ（`data/measures.js`）はリポジトリ内にあるため、**リポジトリのアクセス権 = 施策の見える範囲**。他事業部には見えません。

## 使い始める

GitHub でこのテンプレートの **「Use this template」** から自分の事業部の Private リポジトリを作成し、`data/site.js` を書き換えるだけ。

→ 詳細な手順: **[docs/新しい事業部でこのサイトを使う.md](docs/新しい事業部でこのサイトを使う.md)**

## 見る

`index.html` をブラウザで開く。施策0件でも動きます。

> Chrome の `file://` で PDF取り込みが動かない場合は Firefox で開くか、`python3 -m http.server` して `http://localhost:8000` で開いてください。

## 施策を追加する

- **おすすめ（綺麗なカード）**: PDFをリポジトリに置き、Claude Codeに読ませて生成 → **[docs/PDFからカードを作る_Claudeへの指示.md](docs/PDFからカードを作る_Claudeへの指示.md)**
- **お試し（荒い下書き）**: ヘッダー「＋ PDFから取り込み」→ そのブラウザだけに即カード化（共有されない・機械抽出）

> ブラウザのドロップ取り込みはAIではなく正規表現の仕分け機です。内容を理解した綺麗なカードは Claude Code に読ませて作ります。

データ構造: **[docs/施策データの書き方.md](docs/施策データの書き方.md)**

## 技術メモ

- 依存ゼロの静的サイト（`fetch()`/`type="module"` 不使用 = `file://` で動く）
- pdf.js は `vendor/` に同梱（CDN不使用）
