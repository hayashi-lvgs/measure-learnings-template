# CLAUDE.md

このリポジトリで作業する Claude Code 向けの入口。**作業開始前に必ずこのファイルと下記ドキュメントを読むこと。**

## このプロジェクトは何か

**施策蓄積サイト（テンプレート）** — 記事/LP等のCRO施策の勝ち/引き分け/負けを蓄積する社内向け静的サイト。**事業部ごとに独立したリポジトリで運用する**ためのテンプレート。

**目的**: 勝ち施策・負け施策から「なぜ勝ったか/負けたか」の要因を言語化して蓄積し、**次のデザイン提案に納得感（根拠・説得力）を持たせる**ための"根拠ライブラリ"。単なる結果アーカイブではない。

3原則（崩さない）:
1. **事実(📋実線)とAI推論・仮説(🤔破線+⚠️)を明示的に区別する**
2. 各施策に**デザイン観点の活かし**(✅有効/⚠️注意/❌NG)を、自事業部のユーザー前提で付ける
3. **横断で学びをパターン化**する

## このリポジトリがテンプレート由来のとき、最初にやること

1. `data/site.js` の `title` / `badge` を事業部用に書き換える
2. `data/measures.js` が空であることを確認（他事業部のデータを持ち込まない＝分離の大原則）
3. 事業部のPDF様式に合わせて `assets/js/extract.js` の `LABELS` を調整（ユーザーに実際のPDFを1枚もらってから）

詳しくは [docs/新しい事業部でこのサイトを使う.md](docs/新しい事業部でこのサイトを使う.md)。

## ファイル構成

```

  measures.js          # ★施策データの正: MEASURES / INSIGHTS / SCREENSHOTS

```

## 作業時の重要ルール

- **正のデータは `data/measures.js`**。1施策の追加は原則ここに `MEASURES` オブジェクトを1つ足すだけで完結（スクショがあれば `SCREENSHOTS` に1行 + `images/` に画像）。
- **`fetch()`／`type="module"` は使わない**。file:// のCORS制限で動かない。データは必ず `<script src="...">`（classic script）で読む。
- **事実とAI推論を混ぜない**。元資料に書かれていない解釈を `factuals` に入れない。
- 新しい指標キーを使ったら `assets/js/app.js` の `METRIC_LABELS` にも日本語ラベルを追記。
- **見た目（CSS）は勝手に変えない**。レイアウトを維持。
- **分離の大原則**: 他事業部のリポジトリから施策データ・画像を持ち込まない。このリポジトリの施策は、このリポジトリのアクセス権を持つ人だけが見る。
- 作業開始時に `git log` とローカル差分を確認。リモートが進んでいたら先に `pull`。
