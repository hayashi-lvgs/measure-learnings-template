施策蓄積サイト 取扱説明書

はじめての人でも使えるように、やさしく説明します。専門知識は要りません。

1. これは何？（30秒でわかる）

やった施策（ABテストなど）の「勝ち・引き分け・負け」と、その理由を貯めていくサイトです。

ただの結果置き場ではありません。「なぜ勝ったのか / なぜ負けたのか」を言葉にして残すことで、次にデザインを提案するときに「前にこういう理由で勝った/負けたから、今回はこうすべき」と根拠を持って話せるようになります。

  • 施策はカードとして一覧表示されます（勝ち 引き分け 負け）
  • 「横断インサイト」では、複数の施策から見えてきた共通パターンが見られます

2. 大事な考え方（3つだけ）

  1. 事実と推測を分ける — PDFに書いてある「事実（📋）」と、AIや人の「推測（🤔）」をハッキリ区別します。ごちゃ混ぜにしません。
  2. デザインへの活かし方を書く — 「✅こうすると効く / ⚠️注意 / ❌やってはいけない」を、自分の事業部のユーザー目線で書きます。
  3. 他の事業部には見えない — このサイトの施策は、このリポジトリ（保管場所）にアクセスできる人だけが見られます。事業部ごとに別々の保管場所を持つので、他事業部の施策は混ざりません。

3. 自分の事業部のサイトを用意する（最初の1回だけ）

すでに用意されたサイトを見るだけの人は、この章はとばして「4. サイトの見方」へ。
  1. GitHub でテンプレートのページを開く
  2. 「Use this template（このテンプレートを使う）」ボタン → 自分の事業部の保管場所（リポジトリ）を作る（公開範囲は Private（非公開））
  3. 作った保管場所を自分のパソコンに取ってくる（git clone）
  4. data/site.js の中の事業部名（badge）を、自分の事業部に書き換える
  5. 一緒に使うメンバーを、GitHubの設定で「Collaborator（共同編集者）」に追加する
パソコン操作やGitに不安があれば、Claude Code にこのフォルダを開いて「セットアップを手伝って」と頼めば、順番に案内してくれます。

4. サイトの見方

開き方

index.html というファイルをダブルクリックすると、ブラウザで開きます。

⚠️ Google Chrome で開いて PDF取り込みが動かないときは、Firefox で開いてください。（Chromeは安全のため、パソコン上のファイルからPDFを読む動作を止めることがあります）

画面の見どころ

  • 一覧: 施策カードが「勝ち / 引き分け / 負け」に分かれて並びます
  • 横断インサイト: 全体の勝率と、複数施策から見えた共通パターン
  • 左のサイドバー: 施策タイプやデバイス（SP/PC）でしぼり込み、キーワード検索
  • カードをクリック: 施策の詳細（概要・仮説・数値・事実/推測・要因・デザイン観点・関連リンク）が開きます

5. 施策を追加する（2つの方法）

✅ 方法A：おすすめ — ClaudeにPDFを読ませて、きれいなカードを作る

内容をちゃんと理解した、共有できる正式なカードを作る方法です。

  1. 検証シートのPDFを、このフォルダの中（例 pdf/ フォルダ）に置く
  2. このフォルダで Claude Code を開く
  3. docs/PDFからカードを作る_Claudeへの指示.md にあるプロンプトをコピーして貼る
  4. Claudeがまず「PDFを読んで理解した内容」を見せてくれるので、合っているか確認する
  5. OKすれば、Claudeがきれいなカードをサイトに追加してくれる
  6. 最後に保存して全員に共有（git commit → git push。ここもClaudeに頼めます）

🔧 方法B：お試し — ブラウザにPDFをドロップ（下書きだけ）

サッと下書きを見たいとき用。きれいにはなりません。

  • サイト右上の「＋ PDFから取り込み」にPDFをドロップ → その場でカードが出ます
  • ⚠️ これは AIではなく機械的な仕分けなので、文章が混ざったり項目が抜けたりします
  • ⚠️ さらに 自分のブラウザにしか残りません（他の人には見えない・パソコンを変えると消える）
  • 正式に共有したいときは、必ず方法Aで仕上げてください
なぜブラウザだけでは綺麗にならないの？ → サイトの中には「考えるAI」が入っていないからです。中身を理解して整理するのはClaudeの役目です。

6. 施策の結果の見方（注意）

カードの「勝ち / 負け」は、そのテスト期間中の結果にすぎません。数字だけで施策の価値を決めつけないでください。季節・ユーザー層・市場の動き・競合など、外の要因も一緒に考える必要があります。

7. 困ったときは（Q&A）

Q. 他の事業部に、うちの施策は見えてしまう？
A. 見えません。施策データは事業部ごとの別々の保管場所にあり、そこにアクセスできる人だけが見られます。

Q. ブラウザにドロップしたカードが消えた
A. ドロップしたカードは「自分のブラウザだけの一時保存」です。共有・保存したいものは方法Aで正式に追加してください。

Q. Chromeで「＋ PDFから取り込み」が反応しない / エラーになる
A. Firefoxで開いてください。それでも困ったらClaudeに「ローカルサーバーで開く方法を教えて」と聞けば案内します。

Q. PDFの項目がうまく読み取れない
A. 事業部によって検証シートの様式が違うためです。方法A（Claude）ならその場で理解して吸収します。ブラウザ取り込みを直したい場合は、Claudeに「うちのシートに合わせて extract.js を調整して」と頼めます。

Q. 操作がわからない
A. このフォルダで Claude Code を開き、やりたいことを日本語で伝えてください。このサイトの仕組みはClaudeが理解しています。
