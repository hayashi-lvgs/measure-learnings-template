# 新しい事業部でこのサイトを使う（立ち上げ手順）

このリポジトリは **施策学習サイトのテンプレート（空）** です。事業部ごとに独立したリポジトリを1つ作って使います。

## なぜ事業部ごとにリポジトリを分けるのか

- **他事業部に施策を見せない**ため。施策データ（`data/measures.js`）はリポジトリ内にあるので、リポジトリのアクセス権限＝施策の見える範囲になる。
- テンプレートから作ったリポジトリは**フォークではなく完全独立**。あなた（作成者）の事業部のアカウント/org配下にでき、他事業部やテンプレート元にはぶら下がらない。
- **絶対にやってはいけないこと**: 他事業部のリポジトリから `data/measures.js` や `images/` をコピーしてくること。分離が崩れる。

## セットアップ（5分）

### 1. 自分の事業部のリポジトリを作る
GitHub でこのテンプレートリポジトリを開き、**「Use this template」→「Create a new repository」**。
- オーナー: **自分の事業部のアカウント/org** を選ぶ
- Visibility: **Private**
- 名前: 例 `xxx-measure-learnings`

> `gh` CLI なら: `gh repo create <org>/xxx-measure-learnings --private --template <テンプレートのowner>/<テンプレート名>`

### 2. clone する
```
git clone https://github.com/<org>/xxx-measure-learnings.git
cd xxx-measure-learnings
```

### 3. サイト名を設定する
`data/site.js` を開き、`title` と `badge` を自分の事業部用に書き換える。ここだけでヘッダーとタブ名が変わる。

### 4. 表示を確認する
`index.html` をブラウザで開く。施策0件でもサイトは正常に動く（空の状態が出る）。

> ⚠️ Chrome で `file://` 直開きだと PDF取り込みの worker が起動しないことがある。その場合は Firefox で開くか、フォルダで `python3 -m http.server` してから `http://localhost:8000` で開く。

### 5. 事業部メンバーを追加する
GitHub のリポジトリ設定 → Collaborators で、同じ事業部のメンバーを追加。これで**その事業部内だけ**で共有される。

## 施策を追加する

### A. お試し（自分のブラウザだけ・共有されない）
ヘッダーの「＋ PDFから取り込み」に検証用シートPDFをドロップ → その場でカード化。ただし保存先はそのブラウザのlocalStorageだけで、**他の人には見えない／端末を変えると消える**。

### B. 事業部の全員に共有する（正式）
1. `tools/import.html` をブラウザで開き、PDFをドロップ → スニペット生成
2. `data/measures.js` の `MEASURES` 配列末尾に貼り付け
3. **内容をレビューして仕上げる**（勝ち/負け判定・事実/AI推論の仕分け・デザイン観点。詳細は [施策データの書き方.md](施策データの書き方.md)）
4. `git add -A && git commit && git push`

→ これで初めてリポジトリを見る全員（＝その事業部）に共有される。

## 自分の事業部のPDF様式に合わせる（重要）

PDF抽出ロジック `assets/js/extract.js` は、ある「検証用シート」様式のラベル（施策No./仮説/検定結果…）前提でチューニングされている。**事業部で違う様式のシートを使っている場合、そのままではうまく項目を抜けない。**

対処: このリポジトリで **Claude Code を開き**、「うちの検証シートはこういう項目名だから `extract.js` の `LABELS` をこう変えて」と依頼すると調整できる。まずは実際のPDFを1枚ドロップして、何が抜けて何が抜けないかを見てから調整するとよい。

## （任意）URLで配布したい場合
社内に見せたいなら GitHub Pages を有効化する（Settings → Pages → Branch: main）。Private リポジトリの Pages は、GitHub の権限を持つ人だけがアクセスできる（プランによる）。fetch は使っていないので Pages 上でもそのまま動く。
